🤖 AI in Browser

Chrome 138正式推出了三个重要的AI API：Translator API（翻译）、Language Detector API（语言检测）和Summarizer API（文本摘要），均基于设备端AI模型，为开发者提供了强大的本地AI能力。

📱 Device and Sensors

新增Web Serial over Bluetooth支持（Android平台）和Viewport Segments Enumeration API，为可折叠设备和蓝牙串口通信提供了重要支持。

Current Stable Version (Chrome 138)

• Bookmarks and reading list improvements - 对于Chrome 138桌面版，一些在保存新书签时登录Chrome的用户现在可以在其Google账户中使用和保存书签及阅读列表项目。相关的企业策略继续按以前的方式工作，因此管理员可以配置用户是否可以在其Google账户中使用和保存项目。

User Productivity Updates on Chrome Desktop

Current Desktop Productivity Features (Chrome 138)

• Agentspace recommendations正式发布 - 为帮助企业用户满足内部信息需求，现在可以将来自Agentspace的企业搜索结果（如人员、文件或查询建议）添加到Chrome地址栏和新标签页搜索框中。结果可以默认显示或仅在自定义关键字触发时显示。

• AI Mode搜索推荐开始推出 - AI模式是一个功能，通过在Chrome中显示AI模式搜索推荐，帮助用户深入探索关心的主题。新策略AIModeSettings可用于控制地址栏和新标签页搜索框中的搜索推荐。

• 上下文搜索建议功能 - 通过此功能，用户可以直接在当前页面的上下文中询问任何问题。基于地址栏现有的搜索习惯，用户可以通过选择屏幕上的任何内容或用文字提问来使用Google Lens提问。

• 共享标签页组功能推出 - 用户现在可以使用共享标签页组功能在标签页上进行协作。Chrome 138推出加入和使用共享标签页组的能力。稳定版Chrome用户将无法创建共享标签页组（入口点不可用）。

• 语言检测器API - 语言检测器API是一个用于检测文本语言并提供置信度级别的JavaScript API。这可以与翻译功能结合使用，例如接受未知语言的用户输入并将其翻译为特定目标语言。

• 摘要生成器API - 摘要生成器API是一个用于生成输入文本摘要的JavaScript API，由AI语言模型支持。通过公开这个内置模型，避免每个网站都需要下载自己的多吉字节语言模型。

• 翻译器API - 翻译器API是一个为网页提供语言翻译功能的JavaScript API。当浏览器的内置翻译能力无法提供帮助时，这对Web开发人员特别有用。

Upcoming Productivity Changes

• Gemini在Chrome中的扩展 - Chrome 139开始，Gemini功能将在美国为已登录Chrome的用户逐步推出稳定版。

• AI模式移动平台支持 - Chrome 139将在Android和iOS的地址栏中开始推出AI模式功能。

Chrome Mobile Security Updates (Android/iOS)

Current Mobile Security Features (Chrome 138)

• Enhanced Safe Browsing同步功能 - 在Chrome 138中，Chrome的增强安全浏览成为同步功能。这意味着如果用户在一台设备上选择启用增强安全浏览，此保护级别将自动应用到已登录同一账户的所有其他设备上。

• DLP文件系统访问API支持 - 数据丢失防护保护现在覆盖使用文件系统访问API下载的文件和目录。此增强确保现代Web应用程序（如基于浏览器的编辑器）的下载按照组织的DLP规则进行扫描。

• 客户端LLM反诈骗辅助 - 为对抗诈骗，Chrome现在使用设备端LLM为增强安全浏览用户识别诈骗网站。Chrome将页面内容发送到设备端LLM以推断页面的安全相关信号，并将这些信号发送到安全浏览服务器以获得最终判断。

• iPad上的Google Lens屏幕搜索 - 将iOS上的"使用Google Lens搜索屏幕"功能扩展到iPad设备。iPad通常与更复杂的任务相关，例如购物，在iPad上扩展Lens功能使用户能够轻松执行这些任务。

• Android蓝牙串行通信 - 此功能允许网页和Web应用程序在Android设备上通过蓝牙连接到串行端口。Chrome on Android现在支持通过蓝牙RFCOMM的Web Serial API。

Upcoming Mobile Security Changes

• 恶意APK下载检查 - Chrome 139开始，Chrome on Android将就在Chrome中下载的APK文件联系Google服务器，以获得关于其安全性的判断。

• Manifest V3迁移截止日期 - 2025年6月之前，扩展程序必须更新为利用Manifest V3。Chrome 139将移除ExtensionManifestV2Availability策略。

• CA证书变更 - Chrome 139对中华电信和Netlock颁发的公开信任TLS服务器认证证书的默认信任方式进行更改。

Chrome 138代表了Google在企业功能方面的重大进步，特别是在多平台统一管理和AI集成方面。与之前版本相比，此版本的关键发展趋势包括：

配置文件管理现代化 - iOS多身份支持和改进的书签同步功能表明Google正在统一跨平台的用户体验，同时保持企业级数据分离。
